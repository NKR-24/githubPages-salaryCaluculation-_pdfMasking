# 明細書マスキング（ブラウザ内処理）

保険代理店向けの手数料明細PDFから、契約者名などをブラウザ内で黒塗りするツールです。
**PDFはどこにも送信されません。** 処理はすべて利用者のブラウザ内で完結します。

## 対応様式

| アップ欄 | 様式 | 黒塗りする箇所 |
|---|---|---|
| A: あいおい生命 | 手数料支払明細書（三井住友海上あいおい生命） | 会社名の行 ＋ 契約者名の列（カナ行含む） |
| B: 三井住友海上 | 手数料早期支払明細書（三井住友海上） | 会社名の行 ＋ 契約者名の列（内容不問・全行） |

間違った欄にアップした場合は様式チェックで検知し、処理を中止してエラー表示します。

## Claude連携用プロンプト

アップ欄の下に、黒塗り後のPDFをClaudeでExcel化するための定型プロンプトが
様式ごとに用意されています（コピーボタン付き）。文面はページ上で編集できますが、
再読み込みで元に戻ります。恒久的に変えたい場合は `index.html` 内の
`<textarea id="prompt-aioi">` / `<textarea id="prompt-ms">` の中身を書き換えてください。

## 仕組みと安全性

各ページを一度「画像」に変換し、様式ごとの固定座標＋テキスト位置情報で
契約者名の列を黒く塗りつぶしてから、画像だけのPDFとして再構成します。
**テキスト層そのものが存在しなくなる**ため、黒塗りの下から文字をコピー・
検索・抽出される余地が原理的にありません（AIやOCRにかけても黒い矩形しか見えません）。

## 制限・注意

- 出力は画像ベースのPDFです（残した部分もテキスト検索不可・ファイルサイズ増）。
- 1ファイルあたり最大 **1000ページ** まで対応します（超過時はエラー表示して中止）。
  - **40ページ以下**（例: あいおい生命 34ページ）: 全ページプレビュー＋手動黒塗り追加・やり直しが使えます。
  - **41ページ以上**（例: 三井住友海上 455ページ）: 省メモリの逐次モードになり、
    先頭3ページのみプレビュー表示・手動追加は不可です（全ページ処理・保存は可能）。
- ページ数が多いと環境により数分、1000ページ級は10分前後かかることがあります。
  処理中はタブを閉じず、PCをスリープさせないでください。
  出力の画像PDFは数十MB規模になることがあります。
- メモリの都合上、大きなファイルは一度に何本もアップせず、1〜2本ずつ処理してください。
- 保険会社が様式（レイアウト）を変更した場合は位置がズレます。
  `index.html` 内の `FORMATS` 定数（単位: PDFポイント）の
  `band`（列の左右端）や `bodyY`（表の上下端）を調整してください。
- 募集人コード・代理店コード・証券番号は黒塗り対象に含めていません。
  必要なら `FORMATS` に帯を追加してください。

## 保存後の最終チェック

出力PDFを開いて、①黒塗り部分をコピーできない、②契約者名・会社名がテキスト検索で
ヒットしない、③黒塗りの掛け漏れがない、を確認してから共有してください。
